# SRS 算法优化研究

## 1. SM-2 算法的问题

当前实现基于 SuperMemo SM-2（1988 年），存在以下局限：

### 1.1 固定的初始间隔
- 第一次：1 天
- 第二次：6 天
- **问题**：不考虑词汇难度差异，所有词一视同仁

### 1.2 简单的评分系统
- 只有 3 档：forgot / hard / good
- **问题**：无法细粒度反馈记忆强度

### 1.3 线性的难度系数调整
- easeFactor 按固定公式调整
- **问题**：不考虑遗忘曲线的非线性特征

### 1.4 忽略复习历史
- 只看当前状态（easeFactor, interval, repetitions）
- **问题**：不考虑长期记忆模式

## 2. FSRS (Free Spaced Repetition Scheduler) 改进点

Anki 在 2023 年引入的 FSRS 算法，基于机器学习优化：

### 2.1 四档评分
- Again（完全忘记）
- Hard（困难）
- Good（良好）
- Easy（容易）

### 2.2 记忆状态建模
- **Stability（稳定性）**：记忆保持时间
- **Difficulty（难度）**：词汇本身的难度
- 分离词汇难度与记忆强度

### 2.3 遗忘曲线拟合
- 基于真实用户数据训练
- 考虑复习间隔的非线性效应

### 2.4 复习历史权重
- 近期复习权重更高
- 长期记忆模式影响未来间隔

## 3. 适合本项目的优化方向

考虑到：
- MVP 阶段，避免过度复杂
- 技术词汇特点（专业性强）
- 用户场景（工作中高频遇到的词需要快速巩固）

**推荐方案：SM-2 改进版（介于 SM-2 和 FSRS 之间）**

### 3.1 保留简单性
- 仍用 3 档评分（降低认知负担）
- 核心算法仍基于 SM-2

### 3.2 关键改进
1. **词汇难度初始化**
   - 根据词长、技术性判断初始 easeFactor
   - 技术术语（如 "idempotent"）初始 easeFactor 降低到 2.2

2. **新词学习阶梯优化**
   - 第一次：10 分钟（即时巩固）
   - 第二次：1 天
   - 第三次：3 天（而非 6 天，更平滑）

3. **遗忘后的惩罚减轻**
   - 不完全重置到 1 天
   - 根据之前的最大间隔调整：`max(1, previousInterval * 0.25)`

4. **Hard 档位的优化**
   - 当前：interval * 0.8，repetitions 不变
   - 优化：interval * 0.7，并且下次更容易触发 Good

5. **添加"连续答错"检测**
   - 连续 3 次 forgot → 标记为"困难词"
   - 困难词的间隔增长更保守（easeFactor 上限 2.0）

## 4. 数据结构变更

```typescript
export interface SRSState {
  easeFactor: number
  interval: number
  repetitions: number
  dueDate: number
  
  // 新增字段（向后兼容）
  lapses?: number              // 遗忘次数（连续答错计数）
  maxInterval?: number         // 历史最大间隔（用于遗忘后恢复）
  lastReviewed?: number        // 上次复习时间（用于分析）
  initialDifficulty?: number   // 初始难度（1-5，用于统计分析）
}
```

## 5. 迁移策略

- 所有新字段都是 optional
- 旧卡片自动补充默认值
- 不影响现有用户数据

## 6. 预期效果

- **新词巩固更快**：10 分钟 → 1 天 → 3 天（平滑过渡）
- **遗忘惩罚更轻**：不完全重置，减少挫败感
- **困难词识别**：自动降低增长速度，避免频繁遗忘
- **技术词汇适配**：长词/术语自动降低初始 easeFactor

## 7. 是否采用完整 FSRS？

**不推荐**，理由：
1. **复杂度高**：需要引入 17 个参数的神经网络模型
2. **训练数据缺失**：本项目用户基数小，无法训练有效模型
3. **边际收益低**：SM-2 改进版已能满足 90% 需求
4. **维护成本**：FSRS 算法仍在快速迭代，API 不稳定

**结论**：采用 SM-2 改进版（上述 3.2 的 5 点改进）
