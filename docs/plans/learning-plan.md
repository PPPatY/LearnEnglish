# LearnEnglish 学习方案

## 目标定位

从"能说简单日常对话"（A2-B1 水平）突破到"与外国同事流畅技术交流"（B2-C1 工作场景）。

核心突破点：
- **输出跟上输入**：不只是"听懂"和"看懂"，更要能"说出来"和"写出来"
- **技术场景肌肉记忆**：standup、code review、技术争论、PR 描述等场景的高频表达
- **中式英语 → 地道表达**：在真实输出中发现问题、即时纠正、形成记忆

## 每日 90 分钟结构

| 时段 | 模块 | 能力 | 工具/方法 |
|------|------|------|-----------|
| **15 min** | 词汇 SRS | 词汇量 | 本应用：复习到期词卡，评分驱动下次出现 |
| **20 min** | 写作批改 | 技术写作 | 本应用：写 PR/邮件/issue，AI 批改语法和地道度 |
| **30 min** | 口语陪练 | 口语/听力 | 本应用：语音对话，AI 角色扮演（standup/code review/面试） |
| **20 min** | 听力精听 | 听力/语音 | 外部资源：技术播客片段 + 跟读（见下方推荐） |
| **5 min** | 复盘 | 全部 | 记录今天的高频错误、新学的地道表达 |

**总计**：90 分钟

## 为什么这样安排？

### 1. 输入+输出双轮驱动

- **输入**（词汇 SRS + 听力）：扩展词库、熟悉语音
- **输出**（写作 + 口语）：逼自己调用、在真实场景中暴露问题
- **即时反馈**（AI 批改 + 反馈报告）：不是埋头苦练，而是每次都知道哪里错了、怎么改

### 2. 技术场景导向

通用英语教材对工程师帮助有限 —— 你不需要在酒店前台订房的表达，你需要能说清楚"这个 API 是幂等的"。

本工具的所有任务和场景都围绕**技术工作场景**：
- 词汇库：50+ 技术词汇（deprecate / idempotent / throttle / race condition...）
- 写作任务：PR 描述、技术邮件、issue 回复、函数注释
- 口语场景：standup、code review、技术讨论、系统设计面试

### 3. 碎片化 vs 系统化

- **碎片时间**（通勤、午休）：词汇 SRS（15 分钟）
- **整块时间**（晚上、周末）：写作 + 口语（50 分钟）
- **长期坚持**：每天 90 分钟比"今天学 3 小时、后天不学"有效得多

## 如何使用本工具

### 词汇 SRS

1. 打开"🃏 词汇 SRS"模块
2. 点击"复习"，开始翻卡片
3. 看到单词 → 回想释义 → 点击翻面 → 评分（忘了/模糊/认识）
4. 系统自动调整下次复习时间：
   - 认识 → 间隔拉长（1 天 → 6 天 → 更久）
   - 忘了 → 重新开始（明天再来）
5. **加生词**：从写作批改或口语反馈中发现的高频错误词，手动加入

**目标**：每天复习完所有到期的卡片（通常 10-20 个）

### 写作批改

1. 打开"✍️ 写作批改"模块
2. 选择任务（如"PR 描述"）
3. 用英文写（不要担心错误，这就是训练的意义）
4. 提交批改 → AI 返回：
   - **语法错误**（时态、主谓一致、拼写）
   - **地道度建议**（中式英语识别、更自然的说法）
   - **完整改写版本**（对比学习）
5. 对比原文和改写，找出自己的习惯性错误

**目标**：每天写 1-2 个任务，积累真实场景的表达

### 口语陪练

1. 打开"🎙️ 口语陪练"模块
2. 选择场景（如"Daily standup"）
3. AI 开场（如："What did you work on yesterday?"）
4. 点击麦克风 🎙️ → 用英文回答 → AI 识别并回复
5. 多轮对话（5-10 轮）
6. 结束时 AI 生成反馈报告：
   - 总评
   - 建议改进的地方
   - 表现好的地方

**目标**：每天练 1-2 个场景，每个场景至少 5 轮对话

**注意**：
- 使用 **Chrome 或 Edge** 浏览器（语音识别支持最好）
- 安静环境（降低识别错误）
- 如果识别不准，可以手动输入文字（后续版本会支持）

### 听力精听（外部资源）

本工具暂未内置听力模块，推荐外部资源：

**技术播客**（英文原声，有字幕更佳）：
- [Syntax.fm](https://syntax.fm/) —— 前端技术讨论
- [Software Engineering Daily](https://softwareengineeringdaily.com/) —— 工程实践访谈
- [The Changelog](https://changelog.com/podcast) —— 开源项目深度聊

**精听方法**：
1. 选一段 5-10 分钟的片段
2. 听第一遍，抓大意
3. 听第二遍，逐句暂停，写下听到的内容
4. 对照字幕，找出没听清的地方
5. 跟读（shadowing）：边听边模仿语音语调

**目标**：每天 20 分钟，一周内把一段听到"不看字幕也能懂 80%"

## 每日复盘（5 分钟）

在一个笔记本或文件里，记录：
- 今天的高频错误（时态、介词、词汇混淆等）
- 新学的地道表达（从批改和反馈中提取）
- 明天重点关注的 1-2 个问题

**示例**：
```
2024-01-15
- 错误：总是忘记第三人称单数的 -s（"he work" → "he works"）
- 地道表达：说"我昨天修了个 bug"不是"I repaired a bug"，而是"I fixed a bug"
- 明天重点：注意动词的第三人称单数
```

## 如何坚持？

### 1. 固定时间

把学习绑定到一个固定时间槽：
- 早上通勤：词汇 SRS（15 分钟）
- 晚上 8-9 点：写作 + 口语（50 分钟）
- 睡前：听力精听（20 分钟）

### 2. 小步快跑

不要一开始就强迫自己每天 90 分钟 —— 先从"每天 30 分钟，词汇 + 写作"开始，养成习惯后再加口语和听力。

### 3. 可见进度

本工具的首页 Dashboard 会显示：
- 词库数量、待复习词数
- 批改次数、对话次数

看到数字增长是很强的激励。

### 4. 真实场景

把工具练习和实际工作结合：
- 写 PR 时先在工具里练一遍
- Standup 前用工具模拟一次
- 看到 issue 用英文回复时，先在工具里批改草稿

**真实需求是最好的驱动力**。

## 进阶路线

### 3 个月目标

- **词汇**：掌握 100+ 技术词汇，能快速调用
- **写作**：能写清晰的 PR 描述和技术邮件，不再有明显的中式英语
- **口语**：standup、code review 时不再卡壳，能流畅表达

### 6 个月目标

- **词汇**：200+ 技术词汇，覆盖常见技术场景
- **写作**：能写复杂的技术文档（设计文档、API 文档）
- **口语**：能参与技术争论、系统设计讨论，表达清晰有逻辑

### 1 年目标

- **全面突破**：与外国同事交流无障碍，能承担技术面试（面试官角色）

## 常见问题

**Q1：我没有 API key 怎么办？**  
A：词汇 SRS 模块不需要 AI，可以先从这个开始。写作和口语需要 Claude API key（参考 README 配置）。

**Q2：语音识别不准怎么办？**  
A：确保使用 Chrome/Edge 浏览器，安静环境，说话清晰。未来版本会支持手动输入兜底。

**Q3：每天 90 分钟太多了？**  
A：先从 30 分钟开始（词汇 15min + 写作 15min），习惯后再加其他模块。

**Q4：学了一段时间没感觉进步？**  
A：进步是渐进的，不是线性的。坚持 1 个月后回头看第一周的写作/对话记录，你会惊讶于自己的变化。

**Q5：有没有学习小组或社区？**  
A：目前本工具是单机版，未来会考虑加入社区功能（如每周学习打卡、互相批改）。
